# Mica v0.2.0

本地优先(local-first)统一:云文档现在也能离线读写,web 端也有了本地库。

这是 Mica 至今最大的一次架构升级——把原来「在线云同步」和「本地离线」两个割裂的世界,合并成**一套本地优先存储 + 可选云同步**(对标 AFFiNE / AppFlowy),云笔记从此断网也能开、能改,联网自动对账。

## 主要变更

- **云文档离线可读可写**:打开过的云笔记会镜像到设备本地。之后断网、重启、服务器不可达,照样能进工作区、翻页树、打开并**编辑**已缓存的云笔记;联网后本地改动自动推送、远端改动自动合并(CRDT,不丢不撞)。以前离线只能干等。
- **双模式溶解为一套**:不再有「云模式 / 本地模式」的切换。现在是一个统一的工作区列表,每个工作区标着它属于本地还是某个云服务器;登录变成一个动作而非一种模式。可以把本地工作区**搬上云**,也可以把云工作区**转成本地副本**(detach)。
- **web 端也 local-first**:web 版现在用浏览器的 IndexedDB 做本地库(文档副本 + 未推送编辑队列 + 页树缓存)。断网刷新页面,已打开过的云文档照样渲染。以前 web 端纯在线,断网即白屏。
- **更省流量的重连**:超陈旧的副本重连时,服务器只回传「你缺的那一小段」增量,而不是整份文档重下(state-vector 快对账)。大文档、久离线后重连明显更快。

## 底层加固(不直接可见,但更耐用)

- 云端持久化改为纯增量 append-log:每次编辑/远端更新落盘从「400ms 后」变「即刻」,崩溃窗口消失;日志定期自动压实,不无限膨胀。
- 一轮多代理对抗复审 + 修复:堵掉一个会导致「重连时静默丢内容」的高危竞态,以及 web 多标签页、存储损坏等平台特有的失败模式。

## 兼容性

- **老用户无感升级**:已登录 / 已配置服务器的用户,升级后照常连原来的服务器,数据都在。
- 客户端与服务端的同步协议**双向后向兼容**:新客户端连旧服务器、旧客户端连新服务器都正常。

## 自动更新

本版可从 v0.1.7 自动更新(设置 → About → 检查更新)。
